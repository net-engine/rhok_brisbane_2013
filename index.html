What is it?
Terrain is a collection of useful, structural stylesheets for your web projects. It leans heavily on the best parts of Sass/SCSS to reduce repetition between projects.
What it's not
Terrain is not a front-end framework built for prototyping, though it can used that way. It differs from popular frameworks such as Bootstrap or Foundation by being much lighter, and aims to stay that way. It's much closer to the wonderful Skeleton, though tailored more towards modern web app UI.

Terrain is…

  • Easily configurable.
  • Expertly crafted.
  • Responsive.
  • Fast to set up.

Components

Extend Terrain with components. Components are standalone modules which we find ourselves using again and again. The go beyond providing a default style for basic elements, and must be included manually.

This is where Terrain goes into the realm of other front-end frameworks, but we only include here what we use unaltered in at least two projects. Keeping these in the project helps to maintain naming consistency.
